PrePrepare Phase

Steps

This is the first phase out of 3 phases of PBFT. It is called pre-prepare or proposal phase, the current primary node (R1) announces the next record that the system should agree upon.

This is the second stage, every node in the network (in this case R2) receives pre-prepare message sent by the primary node (R1), every node (again R2) validates the correctness of the proposal and multicasts a "prepare message" to the group.

This is phase number three where every nodes wait until they collect a quorum of (2f+1) prepare messages and publish this observation with a commit message.

Finally, every node wait for a quorum of (2f + 1) commit messages to make sure that enough nodes have recorded the decision.
